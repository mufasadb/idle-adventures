# Decision History

Repo-local decision log. Mirrors the Obsidian vault `Project Ideas/idle adventures/Decisions & Open Questions` and extends it with the POC-contract decisions made on 2026-06-30. Purpose: stop re-deriving *why* from prose.

## Pre-POC (2026-06-29, from the vault)

| # | Decision | Why |
|---|----------|-----|
| D1 | All-TypeScript (engine, web, server) | One rulebook shared by client + server + tests; no two-language drift |
| D2 | Pure engine `reduce(state, action, seed)` | Programmatic testing, LLM-driven play, cross-platform, anti-cheat from one codebase |
| D3 | Programmatic / LLM-playable is a first-class goal | Test changes without driving the UI |
| D4 | Thin, optional server | Don't block the core loop on a round-trip |
| D5 | PWA-first | Build something cool first |
| D6 | Rendering is a decoupled thin layer (Canvas2D default) | Renderer swap is low-stakes because the engine is pure |
| D7 | Simulation owns time, React owns chrome | React's batching model is wrong for real-time |

| D38 | **Passive detail-perception replaces the scout forecast** (9u9.2, 2026-07-06; **supersedes D11's "spyglass pre-computes the exact outcome"**): the `scout` action/`scouted` event and `SCOUT_*` levers are removed. Design: `docs/superpowers/specs/2026-07-06-combat-info-perception-design.md`. | Blind playtest (9u9.2) flagged that scout handed you the resolved verdict (win/lose, exact hp) before committing — you should try and fail, not precompute. Splitting info into *range* (tools) vs *depth* (always qualitative) kills the spoiler while still letting you plan a counter, and teaches the matchup by playing (pre-fight tease → post-fight "that's why"). |

| D23 | Packed food converts to energy **wholly at embark** (no eat action in the POC), but the food stacks **stay on the expedition loadout** as slot ballast for the whole run; `return` (M5) **discards** them — eaten food is never banked back (2026-07-03) | The slots-vs-supplies tension (spec §3) needs packed food to occupy space for the entire trip: if eating freed slots at embark, max-food packing would be strictly optimal — a non-decision. The stacks represent rations being hauled; only `carry` is banked on return, which closes the food-duplication trap for M5 |
| D24 | `Expedition` gains `cleared: {x,y}[]` — POIs consumed this run (2026-07-03): `gather` one-shots a node and records it; M4's `fight` records defeated monsters the same way | Node exhaustion is required (infinite nodes would make routing pointless — park on one node forever) but the grid is regenerated, never stored; the cleared list is the minimal present-state delta between "the map as generated" and "the map as consumed" |
| D25 | POIs carry their yield: `Poi.material` is stamped from `BIOMES[biomeId].materialTable[kind]` **inside `generateGrid`** (2026-07-03); biome material tables filled at M3 (12 distinct materials across 3 biomes) | `gather` needs biome-flavoured yields (cross-biome recipe pulls, M5) without violating D21's "engine never consults the biome after generation" — stamping at generation keeps the biome read inside the generator; gather reads the POI |
| D26 | Run end — `return` (M5) or combat soft-fail (M4) — banks carry + durable loadout (equipment pieces, tools, transport, backpack) + unspent potions via one shared `endExpedition`; **food is never banked back** (2026-07-03) | Gear must survive runs or crafting is pointless; food was converted to energy at embark and its stacks were pure slot ballast (D23), so banking it would duplicate energy. One helper = no drift between the two run-end paths |
| D27 | Materials are **weighted per (biome, node-type)**, not exclusive (2026-07-04): `BIOMES[id].materialTable[nodeType]` becomes a weighted table (`{silver-ore: 6, iron-ore: 2, copper-ore: 1}`); `generateGrid` rolls each POI's material from the map seed and stamps it (D25 stamping preserved). Refines D25's single-material-per-node model | A single material per biome/node made silver *tundra-exclusive* — a hard gate. Weighting makes tundra the *efficient* silver farm without walling it off: cross-biome recipes become a **soft pull** ("tundra's the place to farm silver") rather than a hard wall, which suits a wide shared recipe tree. Biome read stays inside the generator (D21 intact). Churns M1/M3 generation snapshots (expected) |
| D28 | `pack` is a **plan** on the town-side staging loadout; **`embark` validates + debits the bank** (2026-07-04): `pack(slot,itemId)` edits `state.loadout` without touching bank, validating against `bank − plan-reservations` so a plan never exceeds holdings; equipment slots overwrite, food/potion/tool append; rejects when food+potion stacks would exceed `slotCap(backpack)` (bead note e). `embark` gains a reject path (unaffordable plan) and debits the whole loadout from bank (food→energy). No `unpack` action — reduce a mis-planned consumable by embarking and re-planning next town visit | Keeps the settled 9-action contract (no new `unpack`); bank stays the single source of truth (only debited at the commit point); symmetric with `endExpedition` re-banking durables + unspent potions on run end (D26). Accepts one wart — no in-town decrement of planned food/potions — as the cost of the smaller action surface |
| D29 | Legal-action introspection is **split by phase** (M6, 2026-07-04): `townActions(state)` enumerates bank-facing actions (affordable `craft`, `pack` bank-items-into-slots, `embark` on each candidate map) and never reads a map; `expeditionActions(state)` enumerates map + active-inventory actions (`move` to legal neighbours, `gather`/`scout`/`fight` on the current tile, `drop`, always-legal `return`) and never reads the bank; a thin `legalActions(state)` dispatches by phase. All three build candidates then filter via **speculative `reduce`** (pure + cheap) so they can never drift from what `reduce` accepts — incl. future terrain-gating gear, picked up for free | Mirrors the real UX split (home screen reads your stash; field screen reads the map + what you carried in). Keeps `play(seed, actions[])` untouched — these are read-only inspectors, not part of the reduced action stream. Speculative-reduce = one source of truth for legality (no parallel guard logic to maintain) |
| D30 | `action-rejected` reasons are a **closed union** `RejectionReason` (M6, 2026-07-04, bead notes b/e): all reducer rejections draw from one exported union; `gather`'s overloaded `no-node` is split into `no-node` (empty tile) vs `already-cleared` (consumed node) so `legalActions`/UI can distinguish them | Loose `reason: string` gave callers nothing to switch on; a closed union makes rejection handling exhaustive and the `already-cleared`/`no-node` distinction explicit for the field UI and AI |
| D33 | **Monsters block their tile; walking into one forces a fight** (M7-iterate, 2026-07-05): `move` onto a live (uncleared) monster tile resolves combat instead of stepping — win relocates you onto the cleared tile with loot, loss soft-fails the run (D26). No energy cost (combat spends HP). Shared `fightAt` helper backs both the `fight` action (stand & fight) and `move`. The web A* routes AROUND live monsters (blocked), but allows a monster as the click goal so you can deliberately walk in and fight. | Static-but-freely-passable monsters made routing past them a non-choice — the agent playtest confirmed you could ignore every monster. Making them block their tile turns "path through (fight) vs route around" into a real decision (user request). Keeps the 9-action contract — `fight` still works for a monster on your own tile; `move` just gained the walk-in case |
| D32 | **Candidate maps rotate with `GameState.runs`** (M7-iterate, 2026-07-05): town's 3-map offer is derived from `(seed, runs)` where `runs` counts completed expeditions (incremented in `endExpedition`); each return to town rolls a FRESH batch of Perlin maps. `runs` is optional (absent = 0) so old saves and terse test states still typecheck. | Agent playtest found the world was only ever 3 fixed maps per seed (no run counter), so 29% of seeds couldn't craft an iron-pick and full mithril was reachable in ~8%. Rotation makes the world effectively infinite (100% of seeds reach iron/coal/mithril within ~30 visits, first mithril map by ~run 4), so the tier ladder is actually reachable. Also retuned combat (MONSTER_TIER_*_CURVE up, iron `plate-*` defense down) so cheap iron plate no longer floors tier-3 — the steel/mithril climb now has a real payoff |
| D31 | **Tiered progression via a gather-tool tier gate** (M7-iterate, 2026-07-04): reinterpret `TOOL_QUALITY` as a tier; `gather` rejects `tool-too-weak` (new `RejectionReason`) when the best held tool's quality < `MATERIAL_TIER[material]` (new lever, absent = 1). This single gate enforces the whole tech tree — coal/silver (T2) need an iron-pick, mithril (T3) a steel-pick — with **no smelting step, no new action, no combat-math change**. Every vertical is tiered (ore/tools/weapons/armour/consumables/transport/backpack) and gated implicitly by its inputs' tiers; the firm carry squeeze (`STACK_CAP` 10→5, packs 3/5/7) makes food-vs-loot live. Best gear (full mithril plate) still trivializes combat **by design** but sits atop the longest climb. Design: `docs/superpowers/specs/2026-07-04-tiered-progression-carry-squeeze-design.md` | M7 feel-assessment (`docs/m7-feel-assessment.md`) found F1 best-gear-trivializes had no climb behind it, F2 the slot squeeze was too soft, F4 the first upgrade was slow/undifferentiated. User call: keep the trivialization but make it *earned* via cost + gating, not by reshaping combat. A gather-gate (vs a processing chain) keeps D10 instant-craft and the 9-action contract intact while giving a real ladder; the "visible but locked" node is the pull. `legalActions` reflects locks for free via speculative reduce (D29) |
| D37 | **Carry sources stack + food-energy cut** (M7 Phase 2, zhn + ff7, 2026-07-05): carry capacity is now additive via `carryCap(equipment)` = backpack tier + `TRANSPORT_CARRY` (bringing a beast/cart: horse 2 / mule 4 / wagon 6) + `PANNIERS_SLOTS` (a new `panniers` equipment slot, +4, **only with a `BEAST_TRANSPORTS` horse/mule** — a wagon can't wear saddlebags). So a mule+panniers is a hauler, a horse is fast with a little extra room. `panniers` is worn/free/durable (banks back). ff7: `ENERGY_PER_FOOD` 10→8, `FOOD_ENERGY` ration 8 / trail-ration 16 (kept ≥8 per the forage-only tundra floor). | Spec §4.4/§4.5: the backpack was the only carry lever, so transport was a pure speed choice with no carry identity, and one food stack still bought half a map. Making transport add carry (and gating panniers behind a beast) gives the mule a real hauler identity distinct from the horse's speed, and the food cut stops max-food from dominating reach. Sustainability harness re-verified green after both. |
| D36 | **Combat consumables — the battleItems branch** (M7 "decisions real" Phase 2, bzd, 2026-07-05): new `Loadout.battleItems: ItemStack[]` (parallel to potions) + a new `"battle-item"` LoadoutSlot; a `COMBAT_BUFF` lever (`elixir-of-power {damageAdd:2}` from vampire-ash, `warding-draught {mitigationAdd:3}` from troll-hide). `resolveCombat` sums the packed buffs into `dmgOut`/mitigation and **consumes them all at fight start** (`battleItemsAfter: []`); unused ones bank back on return (bank.ts). They cost inventory slots (counted by `consumableSlots`, pqp) and are the exclusive reward for fighting T3 monsters. The two recipes were the deferred half of peu (§4.2). | Spec §4.3: gives combat a reward branch that *only* fighting unlocks, so "do I fight this vampire/troll?" becomes strategic. Sim-verified the payoff: **steel + mithril-sword + 3 greater-potions LOSES the Wyrm bare but WINS with elixir + warding-draught** — a player who fought their way up reaches the finish line without the full mithril grind, while mithril players just win more comfortably (doesn't trivialize the climb, D34). Consumed-at-fight-start (not per-round) keeps them a pack-time commitment that competes for slots and can be "wasted" on the wrong fight — a real routing decision. |
| D35 | **Slot-based carry + just-in-time food** (M7 "decisions real" Phase 2, pqp, 2026-07-05; **supersedes D23**, updates D26): the inventory is now UNIT-based — each food/potion/battleItem unit and **each tool** takes one slot; only loot materials stack (`STACK_CAP`). Food is **no longer converted to energy at embark**; it's carried and eaten just-in-time by `food.digest`, which eats whole units off the front whenever a spend pushes `energy` below `heldFoodEnergy` (invariant: `energy ≥ heldFoodEnergy`). Each eaten unit frees its slot mid-run; **uneaten food banks back on return** (bank.ts), so no duplication. `energy` keeps meaning the TOTAL remaining (unchanged for the UI); the opened-ration surplus (`energy − heldFoodEnergy`) is free and doesn't bank. Caps rebalanced for the ~5× consumable pressure: `BASE_CARRY_SLOTS` 3→6, packs 4/6/8→8/12/16. New `src/engine/food.ts`. | Playtest + spec §4.4: worn gear is free but consumables/tools were nearly free too, so "best of everything + max food" was just correct. Making every consumable and tool cost a visible slot — and food burn down over the run — turns the loadout into a live food-vs-tools-vs-loot tradeoff that's *temporal* (cramped early, roomy late) instead of a pack-time non-decision. Eating whole units (a bitten ration's leftover is lost, not banked) keeps banking dupe-free without fractional-food bookkeeping; the greedy sustainability harness was re-derived and stays green (food now banks back, and burns down to open loot slots mid-run — verified gathering + backpack bootstrap still work). |
| D34 | **The finish line + real combat rewards** (M7 "decisions real" Phase 1, 2026-07-05): a tier-4 **Ancient Wyrm** boss (magic dmg into a plate hide — punishes the plate strategy; `MONSTER_TIER_*_CURVE[4]` = HP 48 / dmg 20) added to `BIOMES.tundra.creatureTable` with **no spawn code** — its lethality is the gate. `LOOT_TABLE` entries gain an optional `chance?`; a new pure `rollLoot(seed, creature, at)` (in `combat.ts`, keyed by `(state.seed, creature, tile, defId)`) rolls drops deterministically in `fightAt` — Wyrm drops `wyrm-scale ×3` always + `dragonheart` @0.2. Every monster part now crafts into gear/food/capstone (peu): boar→ration, raider→trail-ration, werewolf→warg-jerkin, scorpion→scorpion-plate-chest(no coal), troll→large-pack(alt); boss→`dragonscale-cuirass`(plate def 5) + `wyrmfang`(dmg 8, `wyrmbane` ×2-vs-`dragon` affinity). Map pressure: `POI_DENSITY` 12→18 (maps richer than one run harvests) + `BASE_ENERGY_FLOOR`=20 (no-food embark → ~5 actions, not a dead-loop). Design: `docs/superpowers/specs/2026-07-05-decisions-and-goal-design.md` §4.1/§4.2/§4.5. | Playtest (`docs/m7-playtest-findings.md`) found the loop hollow: combat had no reward (route-around always won), so monster-blocking (D33) was pointless, and there was no finish line. Sim-verified the gate lands exactly: full mithril + mithril-sword needs **≥3 greater-potions** (2 = death — the tension); `wyrmfang`'s dragon affinity turns the brutal first kill into a **farmable node** (wins on 1 potion), so chasing the 1/5 `dragonheart` is the "one more run" hook. `rollLoot` lives outside `resolveCombat` because it needs a seed and the fight-math stays pure; `fightAt` reserves carry only for loot that actually rolled. Combat consumables (`elixir-of-power`/`warding-draught` from vampire/troll) deferred to Phase 2 with the `battleItems` mechanic (`bzd`) — an inert craftable would be a playtest trap. |

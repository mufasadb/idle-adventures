# Decision History

Repo-local decision log. Mirrors the Obsidian vault `Project Ideas/idle adventures/Decisions & Open Questions` and extends it with the POC-contract decisions made on 2026-06-30. Purpose: stop re-deriving *why* from prose.

## Pre-POC (2026-06-29, from the vault)

| # | Decision | Why |
|---|----------|-----|
| D1 | All-TypeScript (engine, web, server) | One rulebook shared by client + server + tests; no two-language drift |
| D2 | Pure engine `reduce(state, action, seed)` | Programmatic testing, LLM-driven play, cross-platform, anti-cheat from one codebase |
| D3 | Programmatic / LLM-playable is a first-class goal | Test changes without driving the UI |
| D4 | Thin, optional server | Don't block the core loop on a round-trip |
| D5 | PWA-first | Build something cool first |
| D6 | Rendering is a decoupled thin layer (Canvas2D default) | Renderer swap is low-stakes because the engine is pure |
| D7 | Simulation owns time, React owns chrome | React's batching model is wrong for real-time |

| D43 | **Combat alive — % mitigation, steepened curves, roster completion, engagements** (si7.1, 2026-07-06): incoming damage moves from flat subtraction to **percentage mitigation** — `dmgIn = max(CHIP_DAMAGE_MIN, dmg × MITIGATION_K/(MITIGATION_K + D))` where `D = Σ(defense ÷ matrix)` and `MITIGATION_K` (**6**) is the armour-power dial (higher K = armour matters less); full iron plate lands ≈ −50%, mithril ≈ −70%, but the toll is never floored to chip — fixes M7 F1 (`plate-*` flooring the whole bestiary). `MONSTER_TIER_HP_CURVE`/`MONSTER_TIER_DMG_CURVE` steepen (HP 8/16/28/54, dmg 4/8/14/24) to matchup-scaled toll bands (bare-kit: good matchup ≈13% of base HP, neutral ≈27%, bad ≥40%) — HP reads as a second run-budget, and the tier-4 Wyrm's 54 HP/24 dmg re-pins the D34 3-greater-potion gate. Roster completes the type spread per biome — `shell-beetle` (woodland, melee/plate), `mirage-wisp` (desert, magic/robe), `ice-crab` (tundra, melee/plate); `forest-bandit` goes ranged so woodland has all three incoming types — each drops its own part (`beetle-shell`/`wisp-essence`/`crab-shell`) into a new recipe (`plate-boots`/`fire-staff`/`ration` respectively). `BIOMES[id].creatureTable` goes **weighted** (`Record<string, number>`, same shape as `materialTable`) instead of a uniform pick — tier-1/2 dominate, the tundra `ancient-wyrm` sits at weight 1 of 16 (properly rare, not 1-in-6). Combat itself goes from atomic to **engagements**: `Expedition.combat?: Engagement` holds a live fight; `fight` now runs **one exchange per call** (both sides swing once) instead of resolving the whole bout; `flee` disengages at the cost of one parting hit and resets the monster's HP (no free disengage — the monster remains to fight another day); `quaff` drinks a potion mid-engagement with no exchange (absorbs 82r's manual-potion half); a new `autoQuaff` toggle (mirrors `autoEat`) auto-drinks at `AUTO_POTION_THRESHOLD` inside exchanges. Packed `battleItems` buffs consume at engagement **start**, not per-round. Non-combat actions (`move`, `gather`, `drop`, `drop-map`, `return`, `eat`) reject `engaged` while `state.expedition.combat` is set — this **amends D29's "always-legal `return`"**: with a live engagement, `flee` is now the always-available out, not `return`. The pre-fight forecast moved to the walk-in path banner (was a dead `herePanel` block). Absorbs beads **2g7.6** (combat agency) and the manual-potion half of **82r**; re-verifies the D34 Wyrm gate against the new curves. Spec: `docs/superpowers/specs/2026-07-06-combat-alive-design.md`. | Playtest v2 G1: flat mitigation let a single armour tier floor damage against the ENTIRE bestiary regardless of matchup (M7 F1), and combat was atomic — no mid-fight agency (2g7.6), no "do I keep swinging or bail" tension, and no button between fights for a manual heal (82r). Percentage mitigation keeps armour meaningful (it lowers the toll) without letting it zero out the game; the steeper curves make that toll legible per matchup instead of a flat tax. Splitting `fight` into one-exchange calls turns every round into a live decision (press on, flee, quaff) instead of a single dice roll, and giving `flee` a real cost (parting hit + monster reset) means disengaging is a choice with a price, not a free escape hatch — which is why it has to supersede `return`'s "always legal" guarantee as the one always-available out. Weighting `creatureTable` fixes an accidental balance bug (uniform pick made the tier-4 boss as common as tier-1 trash) while completing the roster gives every biome a full melee/ranged/magic incoming spread to matchup against. |
